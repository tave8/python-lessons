L'ambiente virtuale risolve il problema di avere versioni non compatibili nel tuo sistema o ambiente.

Per sistema o ambiente si intende: 
- il sistema operativo
- l'utente specifico del tuo computer
- se se hai pubblicato il progetto online, dall'ambiente di quel server ecc.


Il problema è risolto con l'ambiente virtuale. L'ambiente virtuale CONGELA una versione specifica di un pacchetto, 
linguaggio, dipendenza, in modo tale che il tuo progetto NON può avere problemi di compatibilità.

Esempi di dipendenze:
- pandas (analisi dati)
- numpy (calcolo scientifico)

Dipendenza = package = libreria = framework = risolve un problema specifico

Se tolgo la dipendenza, il progetto si rompe.
